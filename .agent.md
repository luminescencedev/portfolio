## 🎨 Conventions de Code

### Naming Conventions

- **Composants**: PascalCase (Reveal.tsx, MyComponent.tsx)
- **Fichiers utils**: camelCase (useWindowSize.ts, cn.ts)
- **CSS files**: Même nom que composant (Reveal.css, MyComponent.css)
- **Screens**: PascalCase + "Screen" suffix (HomeScreen.tsx, AboutScreen.tsx)
- **Types/Interfaces**: PascalCase (User, ComponentProps)
- **Constantes**: SCREAMING_SNAKE_CASE (API_URL, MAX_ITEMS)
- **Custom Hooks**: Préfixe "use" + PascalCase (useScrollProgress, useMinimap)
- **Dossiers**: PascalCase pour composants, lowercase pour utils

### TypeScript

- **Interfaces** pour les props de composants
- **Type** pour les unions et types simples
- Imports avec alias `@/` (configuré dans tsconfig.json)
- Typage strict activé

### React

- **Hooks**: Privilégier useRef pour les valeurs qui ne nécessitent pas de re-render
- **Performance**: Utiliser React.memo quand approprié
- **Refs**: Ne jamais accéder aux refs pendant le render (utiliser state)
- **UseEffect**: Nettoyer tous les listeners et animations dans le return

### GSAP

- Toujours utiliser `gsap.registerPlugin()` pour ScrollTrigger
- Préférer `gsap.quickTo()` pour les animations répétitives (scroll)
- Utiliser `gsap.set()` pour les changements instantanés
- Toujours kill les ScrollTriggers dans le cleanup

### Lenis

- Une seule instance par scroller
- Toujours synchroniser avec ScrollTrigger: `lenis.on('scroll', ScrollTrigger.update)`
- Configurer ScrollTrigger.defaults avec le scroller personnalisé
- Cleanup: `lenis.destroy()` et `ScrollTrigger.killAll()`

### Styling

- **Tailwind CSS**: Classes utilitaires en premier
- **CSS Module/Fichier**: Pour styles complexes ou spécifiques au composant
- **Responsiveness**: Mobile-first approach
- **Animations**: GPU-accelerated (transform, opacity)

### Custom Hooks

- **Quand créer**: Logique réutilisée 2+ fois OU complexe (multiple state + effects)
- **Où placer**: `src/utils/` pour hooks génériques, co-localisés avec composant si spécifique
- **Pattern standard**:

  ```typescript
  export const useMyHook = (param?: string) => {
    const [state, setState] = useState();

    useEffect(() => {
      // Setup logic
      return () => {
        // Cleanup logic
      };
    }, [param]);

    return { value: state, setValue: setState };
  };
  ```

- **Export**: Toujours exporter dans index.ts du dossier parent

## ⚡ Optimisations de Performance

### Animations

1. **GPU Acceleration**:

   ```css
   transform: translateZ(0);
   backface-visibility: hidden;
   will-change: transform | opacity;
   ```

2. **GSAP quickTo**: Pour animations scroll-driven (60fps)

3. **Préchargement**: Images préchargées au montage des composants

4. **Transitions courtes**: 0.15s - 0.3s pour rester réactif

### React

1. **Refs vs State**: Utiliser refs pour valeurs non-render
2. **Mémoïsation**: React.memo avec comparaison custom si nécessaire
3. **Event listeners**: Toujours cleanup dans useEffect return

### Images

- Format: JPG pour photos/gradients, PNG pour logos/icônes, SVG pour vectoriel
- Localisation: `/public/Gradient/` pour accès rapide
- Préchargement: Via `new Image()` dans useEffect
- Nommage: PascalCase + numéro (Gradient1.jpg, Gradient2.jpg...)
- Optimisation: Compresser avant ajout (max 500KB par image recommandé)

### Performance Monitoring

**Outils à utiliser**:

- **React DevTools Profiler**: Identifier re-renders inutiles
- **Chrome DevTools**:
  - Performance > FPS meter (viser 60fps constant)
  - Layers > Vérifier GPU acceleration active
  - Network > Throttling pour tester slow 3G
- **Console**: `?performance` dans React DevTools

**Budget performance**:

- Frame time: <16ms par frame (60fps)
- First Contentful Paint: <1.8s
- Time to Interactive: <3.8s
- Scroll lag: 0ms perceptible

## ♿ Accessibilité (A11Y)

### À FAIRE ✅

- **Alt text**: `alt=""` sur images décoratives, `alt="Description précise"` sur images informatives
- **ARIA labels**: `aria-label` sur éléments interactifs sans texte visible
- **Contraste**: Minimum 4.5:1 pour texte normal, 3:1 pour texte large
- **Navigation clavier**: Toujours tester Tab, Enter, Espace, Échap
- **Focus visible**: Outlines sur éléments focusables
- **Reduced motion**: Respecter preference utilisateur
  ```css
  @media (prefers-reduced-motion: reduce) {
    * {
      animation-duration: 0.01ms !important;
      transition-duration: 0.01ms !important;
    }
  }
  ```

### À NE PAS FAIRE ❌

- Bloquer scroll avec `overflow: hidden` sans alternative clavier
- Animations rapides/clignotantes (risque epilepsie)
- Dépendre uniquement de la couleur pour information
- Oublier les états focus/hover/active

## 📱 Mobile & Touch

### Configuration

- **Touch events**: Lenis `touchMultiplier: 2` (défaut)
- **Viewport**: `<meta name="viewport" content="width=device-width, initial-scale=1">`
- **Responsive breakpoints**: Tailwind (sm: 640px, md: 768px, lg: 1024px, xl: 1280px)

### Optimisations Mobile

- **Animations**: Réduire `waveNumber` sur mobile (8-10 au lieu de 12)
- **Images**: Version mobile plus légère si >500KB
- **Touch targets**: Minimum 44x44px (recommandation Apple/Google)
- **Test devices**: iPhone SE (375px), iPad (768px), Desktop (1920px)

### Testing

```bash

2. Sélectionner device ou custom dimensions
3. Throttling réseau + CPU
4. Test touch events
```

## 🚨 Règles Importantes

### À FAIRE ✅

1. **⚠️ TOUJOURS consulter le .agent.md avant toute modification** (architecture, code, optimisation)
2. **⚠️ TOUJOURS mettre à jour le .agent.md après changement majeur** (avec date et version)
3. **Toujours nettoyer les animations/listeners** dans useEffect cleanup
4. **Privilégier useRef pour les valeurs non-render**
5. **Mémoïser les composants lourds** qui reçoivent des props stables
6. **Tester sur scroll rapide** pour détecter les lags
7. **Précharger les assets** pour transitions fluides
8. **Utiliser GPU acceleration** pour les animations
9. **Documenter les formules mathématiques** (sinus, phases, etc.)

### À NE PAS FAIRE ❌

1. **❌ Ne JAMAIS modifier le code sans avoir lu le .agent.md** (risque de break patterns)
2. **Ne jamais accéder aux refs pendant render** (utiliser state + useEffect)
3. **Ne pas créer de tweens dans le scroll handler** (utiliser quickTo)
4. **Ne pas oublier gsap.registerPlugin(ScrollTrigger)**
5. **Ne pas laisser des console.log en production**
6. **Ne pas utiliser setState dans les handlers de scroll** (cause lag)
7. **Ne pas changer l'image sans transition** (visible jump)
8. **Ne pas oublier de kill les ScrollTriggers**
9. **Ne pas oublier de dater les mises à jour du .agent.md** (format: YYYY-MM-DD HH:MM)

**Chrome DevTools**:

1. Performance tab > Record > Scroll > Stop
2. Analyser FPS drops, long tasks
3. Layers tab > Vérifier compositing (GPU)

### ScrollTrigger avec Lenis

```typescript
useEffect(() => {
  const lenis = new Lenis({
    /* config */
  });

  function raf(time: number) {
    lenis.raf(time);
    requestAnimationFrame(raf);
  }
  requestAnimationFrame(raf);

  lenis.on("scroll", ScrollTrigger.update);

  ScrollTrigger.defaults({
    scroller: scrollContainerRef.current,
  });

  return () => {
    lenis.destroy();
    ScrollTrigger.killAll();
  };
}, []);
```

### Transition d'image fluide

```typescript
gsap.to(element, {
  opacity: 0,
  duration: 0.15,
  onComplete: () => {
    element.src = newImage;
    gsap.to(element, { opacity: 1, duration: 0.15 });
  },
});
```
